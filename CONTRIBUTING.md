# Contributing

## Workflow

1. Open an issue before large work.
2. Keep changes focused and reviewable.
3. Add or update tests for behavior changes.
4. Run the repo checks before opening a PR.

## Commit style

- Use concise, imperative commit messages.
- Keep subject lines short and descriptive.

## Local checks

- `cargo test --workspace`
- frontend test command for the active package manager
- Python SDK tests if the package is changed
